# Cornerstone v1.5.0 Release Notes

## Summary

`v1.5.0` adds the Ontology Candidate Review Workflow.

The backend can now review `ConceptCandidate` and `RelationCandidate` objects generated by the v1.4.0 extraction layer. Authorized reviewers can edit, reject, approve, or merge candidates. Approval promotes candidates into official Concepts or official ConceptRelations only when the existing officialization gates pass.

## New endpoints

```http
PATCH /v1/ontology/concept-candidates/{candidateId}
POST  /v1/ontology/concept-candidates/{candidateId}/approve
POST  /v1/ontology/concept-candidates/{candidateId}/reject
POST  /v1/ontology/concept-candidates/{candidateId}/merge
PATCH /v1/ontology/relation-candidates/{candidateId}
POST  /v1/ontology/relation-candidates/{candidateId}/approve
POST  /v1/ontology/relation-candidates/{candidateId}/reject
POST  /v1/ontology/relation-candidates/{candidateId}/merge
```

## Data model changes

ConceptCandidate now records:

```text
reviewedBy
reviewedAt
reviewNote
promotedConceptId
mergedIntoConceptId
```

RelationCandidate now records:

```text
reviewedBy
reviewedAt
reviewNote
promotedRelationId
mergedIntoRelationId
```

Migration:

```text
migrations/versions/0014_ontology_candidate_review_workflow.py
```

## Trust boundary

This release preserves the core rule:

```text
LLM/extractor output remains candidate-only until authorized review.
```

Promotion requires:

```text
- authorized reviewer
- pending candidate
- existing evidence
- reviewed eligible evidence
- officialization gates
```

## Not included

```text
- no external live LLM provider
- no graph visualization UI
- no batch review
- no semantic deduplication
- no graph depth above 1
- no automatic promotion of unreviewed evidence
```
