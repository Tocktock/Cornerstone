# v1.3.0 Release Notes — Ontology Graph Runtime

## Summary

`v1.3.0` adds the first runtime ontology graph serving layer for Cornerstone.

It builds on the `v1.2.1` ontology SSOT contract and introduces:

```text
- ontology search
- depth-1 ontology graph response
- Concept aliases
- graph trust labels
- graph citations
- official/candidate/mixed graph modes
```

## Added

```text
- `GET /v1/ontology/search`
- `GET /v1/ontology/graph`
- `OntologyGraphService`
- `OntologyGraphMode`
- `OntologySearchResponse`
- `OntologyGraphResponse`
- `Concept.aliases`
- `concept_aliases` persistence table
- migration `0012_concept_aliases_ontology_graph`
- ontology integration tests
- v1.3.0 implementation document
- v1.3.0 readiness document
```

## Changed

```text
- Package version: 1.2.1 → 1.3.0
- README current version section
- Release checker expected version and required docs
- Concept creation now accepts optional `aliases`
- Concept creation rejects name/alias overlap with existing Concepts
- Relation taxonomy expanded for ontology graph edges
```

## API examples

```http
GET /v1/ontology/search?q=payment%20settlement
GET /v1/ontology/graph?concept=payment%20settlement&depth=1&mode=official
```

## What this version does not do

```text
- It does not run an LLM.
- It does not create ConceptCandidates.
- It does not create RelationCandidates.
- It does not add manual file upload ingestion.
- It does not infer missing graph edges.
- It does not visualize the graph.
```

## Main product value

Users can now handle the first backend form of an ontology Single Source of Truth:

```text
Concept → direct Relations → supporting EvidenceFragments → trust label → limitations
```

## Next

`v1.3.1` should add manual uploaded text-like data ingestion.

`v1.4.0` should add LLM ontology extraction, but only as candidate output.
