# v2.0.1 Release Notes — Refactor and Domain Boundary Hardening

## Summary

`v2.0.1` is a behavior-preserving patch-level maintenance release. It preserves the `v2.0.0` Ontology SSOT behavior while reducing duplicated code and clarifying domain/service boundaries.

## Changed

```text
- Package/runtime version moved from 2.0.0 to 2.0.1.
- Ontology graph depth policy now comes from the ontology domain module.
- Evidence/provenance helpers are shared by grounded context and ontology graph serving.
- Freshness/citation evaluation rules are shared by grounded and ontology evaluation.
- Sync job claimability is shared by in-memory and SQLAlchemy stores.
- Connector timestamp/hash/API-error helper logic is shared by provider adapters/mappers.
- Live proof integer environment parsing is shared by Notion and Google Drive proof scripts.
- Evaluation task validators delegate to shared helper functions.
- Release checker verifies v2.0.1 docs and version consistency.
```

## Unchanged

```text
- API behavior
- database schema
- migrations
- official graph trust boundary
- graph depth limit
- extraction provider behavior
- candidate review behavior
- operator proof behavior
- SSOT readiness checklist semantics
```
