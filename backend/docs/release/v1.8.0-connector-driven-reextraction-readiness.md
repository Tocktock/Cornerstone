# v1.8.0 Connector-driven re-extraction readiness

## Release goal

`v1.8.0` is ready when connector/manual source changes can safely queue ontology re-extraction runs, and those runs can create candidate-only ontology extraction output without mutating the official ontology graph.

The release exists to connect the source ingestion loop to the ontology candidate loop:

```text
changed source evidence
→ OntologyReExtractionRun
→ OntologyExtractionRun
→ ConceptCandidates / RelationCandidates
→ human review remains required
```

## Scope accepted

```text
[x] Add OntologyReExtractionRun as a persisted work item.
[x] Track sync Artifact creation, reuse, and content change metadata.
[x] Queue re-extraction from manual source sync.
[x] Queue re-extraction from manual text/file upload.
[x] Queue re-extraction from connector sync worker success.
[x] Add API endpoints to create, list, read, and run re-extraction runs.
[x] Running re-extraction creates candidates only.
[x] Re-extraction records linked OntologyExtractionRun ids and candidate counts.
[x] Re-extraction records officialGraphMutated=false.
[x] Add version-specific API and release documentation.
```

## Scope rejected

```text
[x] No automatic Concept approval.
[x] No automatic Relation approval.
[x] No official graph mutation from sync or re-extraction.
[x] No live external LLM provider.
[x] No semantic/vector change detection.
[x] No graph visualization UI.
[x] No candidate auto-merge.
[x] No stale candidate cleanup workflow.
```

## Trust readiness

Required guarantees:

```text
[x] Raw source data remains source material only.
[x] Re-extraction output remains candidate-only.
[x] Official graph mutation requires existing candidate review/officialization gates.
[x] Exact source artifact reuse can skip re-extraction.
[x] The re-extraction audit trail records trigger, source, artifacts, evidence, and extraction run ids.
[x] The officialGraphMutated field remains false for this release.
```

## API readiness

New routes:

```text
POST /v1/ontology/re-extraction-runs
GET  /v1/ontology/re-extraction-runs
GET  /v1/ontology/re-extraction-runs/{runId}
POST /v1/ontology/re-extraction-runs/{runId}/run
```

Extended request/response models:

```text
SyncSourceRequest
ManualTextUploadRequest
CreateSyncJobRequest
SyncJob
SyncSourceResponse
```

5. Run ontology graph regression tests.

## Exit decision

This release is acceptable if re-extraction acts as a queue and candidate-generation bridge only. It must not alter reviewed official Concepts, official ConceptRelations, or official graph responses unless reviewers separately approve candidates through the existing review workflow.
