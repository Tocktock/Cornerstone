# v2.0.1 Refactor Readiness

## Purpose

This Cornerstone release readiness document verifies that `v2.0.1` is a behavior-preserving refactor release with clearer domain boundaries and consistent versioning.

## Readiness checklist

| Check ID | Requirement | Measurement | Status |
|---|---|---|---|
| V201-RDY-01 | Version strings are consistent. | `pyproject.toml`, `__init__.py`, readiness schema, release checker, and tests target `2.0.1`. | complete |
| V201-RDY-02 | Runtime behavior remains unchanged. | No new endpoint, migration, graph depth, extraction, review, or automatic officialization behavior. | complete |
| V201-RDY-03 | Ontology domain policy is centralized. | `cornerstone.domain.ontology` owns depth and core status sets. | complete |
| V201-RDY-04 | Evidence support duplication is removed. | Grounded context and ontology graph use `services.evidence_support`. | complete |
| V201-RDY-05 | Evaluation rule duplication is removed. | Grounded and ontology evaluation share `services.evaluation_rules`. | complete |
| V201-RDY-06 | Store sync claimability duplication is removed. | In-memory and SQLAlchemy stores use `services.sync_jobs`. | complete |
| V201-RDY-07 | Connector helper duplication is removed. | Notion and Google Drive mappers/adapters use provider common helpers. | complete |
| V201-RDY-08 | Live proof helper duplication is removed. | Google Drive and Notion live proof scripts use `verification.env`. | complete |
| V201-RDY-09 | Chronicle is updated. | `docs/48-version-chronicle-and-measurable-checklists-v2.0.0.md` includes `v2.0.1`. | complete |
| V201-RDY-10 | Regression checks pass in the available environment. | Compile, release checker, docs tests, refactor-boundary tests, and ontology regression subset. | complete |
