# v1.3.1 Release Notes — Manual Uploaded Data Ingestion

## Summary

`v1.3.1` adds manual uploaded data ingestion to Cornerstone.

Users can now provide source material directly through:

```text
- multipart UTF-8 text-like file uploads
- JSON pasted/manual text uploads
```

Uploaded content becomes Artifacts and EvidenceFragments using the existing provider-neutral sync pipeline.

## Added

```text
- `POST /v1/manual-sources/{sourceId}/uploads`
- `POST /v1/manual-sources/{sourceId}/uploads/text`
- `ManualTextUpload` and `ManualTextUploadRequest` schemas
- `manual_uploads` service for upload normalization and validation
- manual upload size/count settings
- `python-multipart` runtime dependency
- manual upload integration tests
- v1.3.1 implementation document
- v1.3.1 readiness document
```

## Changed

```text
- Package version: 1.3.0 → 1.3.1
- README current version section
- API contract now documents manual upload endpoints
- Release checker expected version and required docs
- Known limitations now state that manual upload supports UTF-8 text-like content only
- `.env.example` includes manual upload limits
```

## API examples

```bash
curl -X POST "http://localhost:8000/v1/manual-sources/$SOURCE_ID/uploads" \
  -F "files=@settlement.md;type=text/markdown"
```

```bash
curl -X POST "http://localhost:8000/v1/manual-sources/$SOURCE_ID/uploads/text" \
  -H 'Content-Type: application/json' \
  -d '{"objects":[{"title":"Settlement Notes","content":"Settlement is the process of finalizing obligations."}]}'
```

## What this version does not do

```text
- It does not run an LLM.
- It does not create ConceptCandidates.
- It does not create RelationCandidates.
- It does not create official Concepts.
- It does not create official ConceptRelations.
- It does not infer graph edges.
- It does not ingest PDFs, Office documents, images, or binary files.
```

## Main product value

Users can now start Cornerstone with manual uploaded knowledge:

```text
uploaded settlement notes → evidence → review → future ontology extraction/review
```

This reduces dependency on live connectors while preserving the same evidence-backed provenance model.

## Next

`v1.4.0` should add LLM ontology extraction from EvidenceFragments, but only into candidate Concepts and candidate Relations.
