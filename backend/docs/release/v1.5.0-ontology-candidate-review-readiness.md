# Cornerstone v1.5.0 Ontology Candidate Review Readiness

## Release theme

`v1.5.0` implements the candidate review workflow that promotes reviewed ontology candidates into the official Single Source of Truth graph.

## Goal

```text
ConceptCandidates / RelationCandidates
→ edit / reject / approve / merge
→ official Concepts / official ConceptRelations
→ explainable ontology graph
```

## Scope delivered

```text
[x] ConceptCandidate edit endpoint.
[x] ConceptCandidate approval endpoint.
[x] ConceptCandidate rejection endpoint.
[x] ConceptCandidate merge endpoint.
[x] RelationCandidate edit endpoint.
[x] RelationCandidate approval endpoint.
[x] RelationCandidate rejection endpoint.
[x] RelationCandidate merge endpoint.
[x] Candidate review metadata.
[x] Audit events for review actions.
[x] Officialization gate preservation.
[x] Per-version implementation document.
[x] API contract and API freeze updates.
[x] Release notes and known-limitations updates.
```

## Non-goals verified

```text
[x] No external live LLM provider added.
[x] No graph visualization UI added.
[x] No batch review workflow added.
[x] No automatic evidence review added.
[x] No automatic promotion without authorized review.
[x] No graph depth above 1 added.
```

## Trust checks

```text
[x] Approve requires authorized reviewer.
[x] Reject requires authorized reviewer.
[x] Edit requires authorized reviewer.
[x] Merge requires authorized reviewer.
[x] Approve only works for pending candidates.
[x] Reject only works for pending candidates.
[x] Merge only works for pending candidates.
[x] ConceptCandidate approval uses officialize_concept.
[x] RelationCandidate approval uses officialize_concept_relation.
[x] Official graph changes only after successful promotion.
```
