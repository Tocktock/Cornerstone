# Cornerstone v1.7.0 Release Notes

## Summary

`v1.7.0` adds **Ontology Evaluation**, a read-only framework for evaluating whether served ontology graphs meet explicit Single Source of Truth quality gates.

This release evaluates graph responses. It does not generate, approve, promote, or mutate ontology data.

## Added

```text
- OntologyGraphEvalTask schema
- OntologyGraphEvalResult schema
- OntologyGraphEvalMetricSummary schema
- OntologyGraphEvaluationService
- In-memory task/result persistence
- SQLAlchemy task/result persistence models
- Alembic migration 0015_ontology_graph_evaluation
- Ontology evaluation API routes
- Integration tests for ontology evaluation API behavior
- v1.7.0 documentation, readiness notes, and API contract updates
```

## New endpoints

```http
POST /v1/evaluations/ontology/tasks
GET  /v1/evaluations/ontology/tasks
GET  /v1/evaluations/ontology/tasks/{taskId}
POST /v1/evaluations/ontology/tasks/{taskId}/run
POST /v1/evaluations/ontology/run
GET  /v1/evaluations/ontology/results
GET  /v1/evaluations/ontology/results/{resultId}
GET  /v1/evaluations/ontology/summary
```

## Evaluation gates

Ontology graph results now record:

```text
graphFound
graphDepthRespected
nodeRequirementsMet
edgeRequirementsMet
evidenceValid
provenancePresent
trustLabelCorrect
freshnessPolicyRespected
officialGraphSafe
candidateBoundaryRespected
relationIntegrityValid
citationValidityRate
success
failureReasons
```

## Metrics

The ontology evaluation summary reports:

```text
ontologyGraphTaskSuccessRate
evidenceValidityRate
provenanceCoverageRate
citationValidityRate
officialGraphSafetyRate
candidateBoundaryRate
relationIntegrityRate
trustLabelCorrectnessRate
freshnessComplianceRate
```

## Non-goals

```text
- no external LLM provider
- no semantic scoring
- no candidate review or promotion
- no automatic official graph mutation
- no graph visualization UI
- no graph depth above 1
```

## Upgrade notes

PostgreSQL deployments should apply the new migration:

```bash
python -m alembic upgrade head
```

Operators can start with one smoke task for the `settlement` concept and require official trust, reviewed evidence, and at least one official relation.
