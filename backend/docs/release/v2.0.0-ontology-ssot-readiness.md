# Cornerstone v2.0.0 Ontology SSOT Readiness

## Release readiness goal

This document defines the v2.0.0 readiness standard for the ontology Single Source of Truth release.

The release is ready when Cornerstone can prove this sequence for a focus concept such as `Settlement`:

```text
source-backed evidence
→ reviewed evidence
→ reviewed candidate promotion
→ official graph
→ valid citations and provenance
→ successful ontology evaluation
→ read-only SSOT readiness response
```

## Required command checks

```bash
python -m compileall -q src tests scripts
python scripts/check_release_candidate.py
python -m pytest tests/integration/test_ontology_ssot_readiness_api.py -q
python -m pytest tests/unit/test_release_candidate_docs.py -q
```

Recommended in dependency-complete local environments:

```bash
python -m pytest -q
python -m ruff check src tests scripts
python -m mypy src
python -m alembic upgrade head --sql
```

## Required API checks

Before proof data exists:

```http
GET /v1/ontology/ssot/readiness?focusConcept=settlement
```

Expected:

```text
status = failed
officialGraphAvailable = false
recommendedActions is non-empty
```

After a successful ontology proof run:

```http
POST /v1/ontology/proof-runs
```

with:

```json
{
  "focusConcept": "Settlement",
  "reviewer": "reviewer@example.com",
  "confirmMutation": true,
  "runEvaluation": true
}
```

Then:

```http
GET /v1/ontology/ssot/readiness?focusConcept=Settlement&depth=1&mode=official
```

Expected:

```text
status = passed
officialGraphAvailable = true
officialGraphSafe = true
latestEvaluationSuccess = true
required checks all passed
```

## Readiness checklist

```text
source_ingestion_available
official_graph_available
graph_depth_policy_respected
official_graph_safe
evidence_citations_valid
review_provenance_present
candidate_boundary_respected
ontology_evaluation_available
operator_proof_path_available
```

## Blocking failures

The release should be blocked if any of these happen:

```text
- readiness endpoint mutates data
- official graph includes non-official nodes or edges
- graph trust label is official without valid evidence
- candidates appear in official mode
- readiness passes without a successful evaluation result
- graph depth above 1 is served
- proof mutation occurs without confirmMutation=true
```

## Non-blocking limitations

```text
- No frontend graph visualization exists.
- Live external LLM provider is not included.
- Persistent-store verification depends on SQLAlchemy/Alembic availability.
- Readiness is focus-concept specific, not a whole-tenant certification.
```

## Release verdict

v2.0.0 is a stable backend contract release when the checklist passes for the chosen focus concept and the release candidate check confirms that the documentation and package version are aligned.
