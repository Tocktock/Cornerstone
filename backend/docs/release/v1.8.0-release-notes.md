# Cornerstone v1.8.0 release notes

## Summary

`v1.8.0` adds connector-driven ontology re-extraction. Manual uploads, manual syncs, and connector sync jobs can now queue ontology re-extraction runs when they create new or changed Artifacts.

Re-extraction is safe by design:

```text
source change → re-extraction run → extraction candidates → review required → official graph
```

It does not promote candidates or mutate the official graph.

## Added

```text
- OntologyReExtractionRun schema and persistence.
- Re-extraction statuses: queued, running, completed, skipped, failed.
- Re-extraction triggers: connector_sync, scheduled_sync, manual_upload, manual_sync, webhook, manual_request.
- Re-extraction API endpoints for create/list/read/run.
- Artifact change metadata in SyncSourceResponse.
- Re-extraction controls in manual source sync, manual text upload, manual file upload, and sync jobs.
- Sync worker event logging for queued/skipped re-extraction.
- Alembic migration for ontology_reextraction_runs and sync job re-extraction fields.
- Version-specific documentation and readiness notes.
```

## Changed

```text
- Source sync now reports created, reused, and changed Artifact ids.
- Manual upload responses can include ontologyReextractionRunId and ontologyReextractionStatus.
- Sync jobs can carry ontologyFocusConcept for downstream extraction focus.
```

## Unchanged trust boundary

```text
- Re-extraction output remains candidate-only.
- Official Concepts and official ConceptRelations still require review/officialization.
- Official graph mode remains protected from pending candidates.
- No live external LLM provider is introduced.
```

## New endpoints

```http
POST /v1/ontology/re-extraction-runs
GET  /v1/ontology/re-extraction-runs
GET  /v1/ontology/re-extraction-runs/{runId}
POST /v1/ontology/re-extraction-runs/{runId}/run
```

## Example

```bash
curl -X POST "http://localhost:8000/v1/manual-sources/$SOURCE_ID/uploads/text" \
  -H "Content-Type: application/json" \
  -d '{
    "objects": [
      {
        "title": "Settlement Notes",
        "content": "Settlement is the process of finalizing obligations. Clearing precedes settlement."
      }
    ],
    "queueOntologyReExtraction": true,
    "runOntologyReExtractionInline": false,
    "ontologyFocusConcept": "settlement"
  }'
```

Then run the queued re-extraction:

```bash
curl -X POST "http://localhost:8000/v1/ontology/re-extraction-runs/$RUN_ID/run" \
  -H "Content-Type: application/json" \
  -d '{"requestedBy":"operator@example.com"}'
```

## Known limitations

```text
- Change detection is based on source object identity and content hash.
- Re-extraction uses the deterministic/local extractor from v1.4.0.
- Candidate review is still separate and API-driven.
- No UI queue is included.
- Persistent-store migration should be verified in a dependency-complete environment.
```
