# Cornerstone v1.6.0 Release Notes

## Summary

`v1.6.0` adds explainable ontology graph serving. Graph responses now explain why the graph has its trust label, how evidence was selected, how candidate objects are separated from official graph objects, and what review actions may still be needed.

## Added

```text
- Graph-level supportSummary.
- Graph-level candidateSummary.
- Graph-level explanation.
- Node reviewProvenance, supportSummary, and explanation.
- Edge source/target display names, focusDirection, reviewProvenance, supportSummary, and explanation.
- Citation dataSourceId, sourceType, sourceExternalId, reviewedBy, and reviewedAt.
- Neighbor Concept evidence in graph citations.
- GET /v1/ontology/explain endpoint.
```

## Changed

```text
- /v1/ontology/graph is now explanation-rich while preserving existing response fields.
- Graph citations now include visible neighbor Concept evidence when available.
- Limitations include pending-candidate warnings when candidate work exists outside official graph mode.
```

## Not changed

```text
- No live external LLM provider is added.
- No candidates are automatically approved.
- No candidate object appears in official graph mode.
- No depth above 1 is supported.
- No graph UI is added.
```
