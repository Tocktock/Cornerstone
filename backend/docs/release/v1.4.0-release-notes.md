# v1.4.0 Release Notes — LLM Ontology Extraction

## Summary

`v1.4.0` adds ontology extraction runs and candidate ontology persistence.

The new runtime flow is:

```text
EvidenceFragments
→ OntologyExtractionRun
→ ConceptCandidates
→ RelationCandidates
```

This release keeps generated ontology objects pending and reviewable. It does not create official Concepts or official ConceptRelations.

## Example

Input evidence:

```text
Settlement is the process of finalizing obligations. Clearing precedes settlement.
```

Output candidates:

```text
ConceptCandidate: Settlement
ConceptCandidate: Clearing
RelationCandidate: Clearing --precedes--> Settlement
```

All candidates are created with:

```text
status=pending
```

## What this version does not do

```text
- It does not approve candidates.
- It does not reject candidates.
- It does not merge candidates.
- It does not create official Concepts.
- It does not create official ConceptRelations.
- It does not mutate the official graph.
- It does not call a live external LLM.
```

## Main product value

Users can now move from source-backed evidence to a proposed ontology review queue:

```text
uploaded settlement notes → evidence → candidate concepts/relations → future review/promotion
```

## Next

`v1.5.0` should add the human review workflow for candidate approval, rejection, editing, and merge/promotion into official Concepts and ConceptRelations.
