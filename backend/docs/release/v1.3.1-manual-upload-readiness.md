# v1.3.1 Manual Upload Ingestion Readiness

## Release identity

```text
Version: v1.3.1
Theme: Manual uploaded data ingestion
Primary doc: docs/40-manual-upload-ingestion-v1.3.1.md
```

## Goal

Ship manual uploaded data ingestion for UTF-8 text-like files and pasted text.

The release proves:

```text
manual uploaded data → SourceObject → Artifact → EvidenceFragment
```

This lets users seed evidence for ontology work before LLM extraction is added.

## Explicit non-goals

```text
- no LLM extraction
- no ConceptCandidate runtime tables
- no RelationCandidate runtime tables
- no candidate review workflow
- no automatic Concepts
- no automatic ConceptRelations
- no graph edge inference
- no PDF/Office/binary extraction
- no graph visualization UI
```

## Trust boundary confirmation

`v1.3.1` does not alter the official ontology trust boundary:

```text
Uploaded text is source material.
EvidenceFragments require review before official use.
No Concepts or Relations are generated by upload endpoints.
No LLM provider or prompt is introduced.
```

## Operator-facing behavior

Operators can now run:

```bash
curl -X POST "http://localhost:8000/v1/manual-sources/$SOURCE_ID/uploads" \
  -F "files=@settlement.md;type=text/markdown"
```

or:

```bash
curl -X POST "http://localhost:8000/v1/manual-sources/$SOURCE_ID/uploads/text" \
  -H 'Content-Type: application/json' \
  -d '{"objects":[{"title":"Settlement Notes","content":"Settlement is ..."}]}'
```

## Risks

```text
- Only UTF-8 text-like content is accepted.
- PDF/Office parsing is deferred.
- Upload sourceExternalId defaults are simple and filename/title based.
- Very large files are rejected instead of chunked.
- Manual upload does not deduplicate semantically similar documents.
```

## Release decision

The release is ready if the verification commands pass and the generated package includes the v1.3.1 implementation document, release notes, readiness document, tests, and updated API contract.
