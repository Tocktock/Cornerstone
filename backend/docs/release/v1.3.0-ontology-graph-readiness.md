# v1.3.0 Ontology Graph Runtime Readiness

## Release identity

```text
Version: v1.3.0
Theme: Ontology graph runtime contract
Primary doc: docs/39-ontology-graph-runtime-v1.3.0.md
```

## Goal

Ship the first runtime ontology graph API over existing reviewed Cornerstone data.

The release proves:

```text
Existing Concepts and ConceptRelations can be served as a depth-1 explainable graph with citations, freshness, trust labels, and limitations.
```

## Explicit non-goals

```text
- no LLM extraction
- no ConceptCandidate runtime tables
- no RelationCandidate runtime tables
- no candidate review workflow
- no manual file upload endpoint
- no graph visualization UI
- no graph depth above 1
```

## Trust boundary confirmation

`v1.3.0` does not alter the `v1.2.1` trust boundary:

```text
LLM output is still candidate-only.
Official graph output still requires official Concepts, official ConceptRelations, and reviewed evidence.
```

No LLM provider, prompt, candidate schema, or extraction run is introduced in this version.

## Operator-facing behavior

Operators can now query:

```http
GET /v1/ontology/search?q=payment%20settlement
GET /v1/ontology/graph?concept=payment%20settlement&depth=1
```

Default mode is `official`, so returned graph objects are suitable for Single Source of Truth use when `trustLabel=official`.

## Risks

```text
- Search is lexical, not semantic.
- Alias conflicts are checked on create, but merge workflows are deferred.
- Candidate/mixed graph modes may be useful for debugging but are not SSOT output.
- Graph responses depend on reviewers creating Concepts and Relations manually until LLM extraction arrives.
```

## Release decision

The release is ready if the verification commands pass and the generated artifacts include this readiness document, release notes, and the v1.3.0 implementation document.
