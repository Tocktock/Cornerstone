# Cornerstone v1.9.0 Release Notes

## Summary

`v1.9.0` adds an operator-focused end-to-end proof checklist for the ontology Single Source of Truth loop.

The release proves:

```text
manual proof source
→ Artifact
→ EvidenceFragment
→ re-extraction
→ ConceptCandidates / RelationCandidates
→ evidence review
→ candidate approval
→ official graph
→ ontology graph evaluation
```

## Added

```text
- POST /v1/ontology/proof-runs
- OntologyProofService
- proof request/response/checklist schemas
- CLI proof --ontology-loop flag
- CLI proof --confirm-ontology-mutation safety flag
- integration tests for proof dry run, confirmation gate, full loop, and reviewer authorization
- v1.9.0 version document, readiness doc, release notes, and API/limitations updates
```

## Changed

```text
- README current version updated to v1.9.0.
- Release checker now expects v1.9.0 and the v1.9.0 proof documents.
- CLI proof report scope can include ontologyLoop when explicitly requested.
```

## Trust Boundary

Still true:

```text
Raw data is not the Single Source of Truth.
LLM/extractor output is not the Single Source of Truth.
Candidate output is not official truth.
Reviewed official ontology graph is the Single Source of Truth.
```

The proof does not bypass these rules. It uses existing review and officialization services.

## Not Included

```text
- frontend graph UI
- live external LLM provider
- automatic approval of production data
- graph visualization UX
- depth greater than 1
- semantic/vector ontology evaluation
```
