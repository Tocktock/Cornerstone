# Cornerstone v1.7.0 Ontology Evaluation Readiness

## Release scope

`v1.7.0` adds a read-only ontology graph evaluation framework. It lets operators define explicit graph quality tasks, run those tasks against the existing ontology graph service, persist results, and summarize graph quality metrics.

The release does not modify source ingestion, LLM extraction, candidate review, candidate promotion, official graph serving, or graph visualization.

## Product readiness checklist

```text
[x] Ontology graph evaluation task schema exists.
```

## Trust boundary checklist

```text
[x] Evaluation is read-only.
[x] Evaluation never approves ConceptCandidates.
[x] Evaluation never approves RelationCandidates.
[x] Evaluation never creates official Concepts.
[x] Evaluation never creates official ConceptRelations.
[x] Evaluation never rewrites evidence.
[x] Evaluation never marks evidence as reviewed.
[x] Evaluation never calls a live external LLM.
```

## Verification checklist

The intended verification set for `v1.7.0` is:

```bash
python -m compileall -q src tests scripts
python -m pytest tests/integration/test_ontology_evaluations_api.py -q
python -m pytest tests/unit/test_release_candidate_docs.py -q
python scripts/check_release_candidate.py
```

Additional regression coverage should be run in a fully provisioned local environment:

```bash
python -m pytest -q
python -m ruff check .
python -m mypy src
python -m alembic upgrade head --sql
```

## Environment note

The sandbox used for this release does not include SQLAlchemy, Alembic, Ruff, or mypy. In-memory API tests and static release checks can run here, but persistent-store and migration verification require a development environment with the project dependencies installed.
