# v1.6.0 Explainable Graph Serving Readiness

## Release objective

Ship an additive graph-serving enhancement that makes ontology graph responses easier to trust and audit without changing the Single Source of Truth boundary.

## Scope

```text
Included:
- graph-level explanation
- graph-level support summary
- graph-level candidate summary
- node/edge review provenance
- node/edge support summaries
- richer citation provenance
- /v1/ontology/explain endpoint

Excluded:
- LLM provider changes
- candidate auto-approval
- graph visualization UI
- graph depth above 1
- semantic/vector graph expansion
```

## Trust boundary check

```text
[x] Official mode still returns only official Concepts and official ConceptRelations.
[x] Candidate objects are summarized, not promoted.
[x] No LLM call is made during graph serving.
[x] Unsupported graph responses remain unsupported.
[x] Explanations describe limitations instead of hiding them.
```

## API readiness

```text
[x] GET /v1/ontology/graph response shape is additive.
[x] GET /v1/ontology/explain is documented.
[x] Existing graph fields remain present.
[x] New fields use camelCase through existing API model behavior.
[x] Depth remains constrained to 0 or 1.
```

## Verification performed in this sandbox

```text
[x] python -S -m compileall -q src tests scripts
[x] direct service smoke test for official settlement graph explanation
[x] direct service smoke test for pending candidate summary without promotion
```

## Environment limitation

The sandbox's default Python startup hangs on normal site initialization, so pytest/TestClient execution was not reliable in this environment. Verification used `python -S` for compile checks and direct service smoke tests with the project source path and installed site-packages explicitly attached.

## Release decision

`v1.6.0` is ready as an additive graph-serving release if the operator reruns the standard full suite in a normal local/dev environment:

```bash
python -m pytest -q
python scripts/check_release_candidate.py
python -m ruff check src tests scripts
python -m mypy src --show-error-codes --no-color-output --no-incremental
```
