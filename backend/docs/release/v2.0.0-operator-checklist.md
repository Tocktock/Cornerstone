# Cornerstone v2.0.0 Operator Checklist

## Purpose

This checklist gives operators a practical path for proving that the reviewed ontology graph can be used as the Single Source of Truth for a focus concept.

## Checklist

### 1. Start API

```bash
cornerstone api --reload
```

Check:

```bash
curl http://localhost:8000/healthz
```

### 2. Run ontology proof loop

```bash
cornerstone proof run \
  --ontology-loop \
  --confirm-ontology-mutation \
  --ontology-focus-concept Settlement \
  --reviewer reviewer@example.com \
  --base-url http://localhost:8000 \
  --json
```

Expected:

```text
ontology_operator_proof_loop = passed
officialGraphAvailable = true
requiredFailed = 0
approvedConcepts > 0
approvedRelations > 0
```

### 3. Run SSOT readiness check

```bash
cornerstone proof run \
  --ssot-readiness \
  --ontology-focus-concept Settlement \
  --base-url http://localhost:8000 \
  --json
```

Expected:

```text
ontology_ssot_readiness_checklist = passed
```

### 4. Inspect readiness directly

```bash
curl "http://localhost:8000/v1/ontology/ssot/readiness?focusConcept=Settlement&depth=1&mode=official"
```

Expected:

```text
status = passed
officialGraphAvailable = true
officialGraphSafe = true
trustLabel = official
latestEvaluationSuccess = true
```

### 5. Inspect graph

```bash
curl "http://localhost:8000/v1/ontology/graph?concept=Settlement&depth=1&mode=official"
```

Expected:

```text
officialGraphAvailable = true
trustLabel = official
evidence is present
nodes and edges include review provenance
```

## Failure actions

If readiness fails:

```text
source_ingestion_available failed → upload/sync source data.
official_graph_available failed → review evidence and approve candidates.
official_graph_safe failed → inspect non-official nodes/edges or invalid evidence.
evidence_citations_valid failed → fix missing or invalid evidence references.
review_provenance_present failed → complete review/officialization gates.
candidate_boundary_respected failed → prevent candidates from official mode.
ontology_evaluation_available failed → run ontology evaluation for the focus graph.
```

## Safety notes

```text
- The readiness endpoint is read-only.
- The proof loop is mutating only when confirmMutation=true.
- Normal connector sync does not mutate the official graph.
- Pending candidates are not official truth.
- Depth 1 remains the default and maximum release graph depth.
```
