# v2.0.3 Release Notes — Dependency-Complete Verification and CI Hardening

## Summary

`v2.0.3` is a patch-level verification release. It adds a dependency-complete verification harness, a CI workflow, and release checks so Cornerstone can be proven in a fully provisioned environment before adding live LLM or UI features.

## No runtime behavior change

`v2.0.3` does not add endpoints, migrations, graph behavior, candidate behavior, connector behavior, live LLM behavior, or frontend UI behavior.

## Operator action

Use the new plan-only command in minimal environments:

```bash
python scripts/run_dependency_complete_verification.py --plan-only
```

Use the strict command in CI or a fully provisioned local environment:

```bash
RUN_POSTGRES_TESTS=1 \
PERSISTENCE_BACKEND=postgres \
DATABASE_URL='postgresql+psycopg://cornerstone:cornerstone@localhost:5432/cornerstone' \
python scripts/run_dependency_complete_verification.py --strict --confirm-live-db
```
