# v2.4.0 Roadmap — Connector Expansion / Live Connector Proof Hardening

## Version goal

Improve source coverage and live proof reliability for connectors while preserving the candidate-only ontology refresh boundary.

This version should make it easier to prove that manual, Notion, Google Drive, and any newly supported connector paths can feed evidence into the ontology loop safely.

## User value

Operators can trust that source changes from real systems create evidence and candidate-only re-extraction work without silently mutating the official ontology graph.

## Confirmed non-goal

`v2.4.0` must not implement:

```text
- automatic official Concept approval
- automatic official Relation approval
- evidence-review bypass
- frontend UI
- graph depth above 1
- live LLM provider changes unless needed only for proof compatibility
- graph visualization schema changes unless required by previous contract fixes
```

## Domain boundary

| Area | Boundary |
|---|---|
| Connectors | Improve supported object coverage and proof reliability. |
| Source sync | Create Artifacts/EvidenceFragments and queue candidate-only re-extraction when appropriate. |
| Official graph | Remains unchanged by connector sync itself. |
| Live proof | Gated by explicit environment variables and operator-supplied credentials. |
| Secrets | Must never be committed or written to proof artifacts. |

## Candidate connector scope

Connector expansion candidates should be prioritized by evidence quality and proofability:

```text
1. Google Drive PDFs or richer text extraction, if safe and deterministic.
2. Google Sheets text/tabular extraction for selected cells/ranges.
3. GitHub file/content connector skeleton or live proof path.
4. Slack message/thread connector skeleton or live proof path.
```

A connector should not be declared supported unless it can create trustworthy source provenance and predictable EvidenceFragments.

## Exit criteria

`v2.4.0` is complete when live connector proof is clearer and source changes can feed ontology re-extraction safely without weakening the SSOT boundary.
