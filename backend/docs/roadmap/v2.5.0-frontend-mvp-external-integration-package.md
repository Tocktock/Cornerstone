# v2.5.0 Roadmap — Frontend MVP or External Integration Package

## Version goal

Choose and implement the first user-facing consumption layer after the backend ontology SSOT loop is stable.

There are two acceptable paths:

```text
Path A — Frontend MVP
A lightweight UI for source upload, candidate review, graph viewing, evidence citations, evaluation, and SSOT readiness.

Path B — External integration package
A client package or integration adapter that lets another product consume the ontology graph, citations, and readiness checks.
```

The release should choose one path explicitly. It should not attempt both unless scope is deliberately reduced and measurable.

## User value

Users should no longer need to understand raw API calls to see the value of Cornerstone. They should be able to either operate a small UI or integrate the official ontology graph into an existing product/workflow.

## Confirmed non-goal

`v2.5.0` must not implement:

```text
- automatic truth generation
- automatic candidate approval
- evidence-review bypass
- unrestricted graph traversal
- new live LLM behavior unless required only for integration compatibility
- new connector expansion unless required only for demo/proof data
```

## Domain boundary

| Area | Frontend MVP path | External integration package path |
|---|---|---|
| User | Reviewer/operator/end user | Developer/integrator |
| Surface | Web UI or local UI | SDK/client/package/API adapter |
| Scope | Operate existing backend loop | Consume existing backend loop |
| Truth boundary | Display official/candidate distinction | Preserve official/candidate distinction |
| Mutation | Only existing reviewed operations | Only documented API calls |

## Decision criteria

Choose **Frontend MVP** if the main risk is product comprehension and review usability.

Choose **External integration package** if the main risk is adoption by existing tools, agents, or applications.

The chosen path must be documented before implementation begins.

## Verification checklist

```text
[ ] compileall passes for backend and any package code.
[ ] release checker passes.
[ ] chosen path quickstart works from clean setup.
[ ] proof demonstrates source/evidence/candidates/review/official graph/readiness consumption.
[ ] official/candidate boundary is visible in chosen surface.
[ ] unsupported concept state is handled clearly.
[ ] chosen path has no secret leakage.
[ ] non-chosen path is explicitly deferred.
```

## Exit criteria

`v2.5.0` is complete when users can experience Cornerstone value through a UI or integration package without needing to manually inspect low-level backend responses.

## Next-version handoff

Handoff depends on the chosen path:

```text
If Frontend MVP: next release should harden reviewer workflows and graph exploration.
If External integration package: next release should harden SDK stability, examples, and agent/tool integrations.
```
