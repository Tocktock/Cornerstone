# v2.3.0 Roadmap — Graph Visualization Contract

## Version goal

Define and implement a frontend-ready graph visualization contract for the existing explainable ontology graph.

This version should make the graph response easier for a frontend, external integration, or visualization client to render without guessing presentation semantics.

## User value

Users can understand a focus concept such as `Settlement` visually:

```text
Clearing --------precedes--------> Settlement
Reconciliation --validates-------> Settlement
Settlement ------updates---------> Ledger
```

A UI can show official/candidate state, evidence citations, review provenance, stale/conflict indicators, and direct neighbors consistently.

## Confirmed non-goal

`v2.3.0` must not implement:

```text
- frontend UI itself
- graph depth above 1
- graph database migration
- automatic graph layout engine as source of truth
- live LLM provider changes
- candidate review behavior changes
- connector expansion
```

## Domain boundary

| Area | Boundary |
|---|---|
| Backend | Return UI-ready graph presentation metadata. |
| Frontend | Not implemented in this version. |
| Graph logic | Reuse existing ontology graph service and depth policy. |
| Trust | Visual states must reflect actual official/candidate/stale/conflict state. |
| Layout | Provide hints only; frontend may choose final rendering. |

## Required behavior

The graph visualization contract should include:

```text
- node display label
- node category/type
- node status display state
- edge display label
- edge direction relative to focus concept
- visual grouping hints
- citation panel payload
- review provenance panel payload
- stale/conflict/candidate indicators
- recommended empty-state text
- frontend example JSON
```

## Exit criteria

`v2.3.0` is complete when a frontend or external client can render a depth-1 ontology graph from the backend response without reverse-engineering status, direction, citation, or provenance semantics.
