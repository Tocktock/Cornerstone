# v0.13.1 Blocker Fix and Verification

## Purpose

This document records why `cornerstone-backend` could not be tagged as
`v1.0.0-rc.1` directly from `v0.13.0`, what was patched in `v0.13.1`, and which
release checks passed after the patch.

## Release Decision

`v0.13.0` was not tagged as `v1.0.0-rc.1`.

Reason:

```text
The original instruction allowed v1.0.0-rc.1 only if v0.13.0 passed
docs/release/backend-release-checklist.md without code changes.
v0.13.0 did not pass unchanged.
```

The narrow blocker patch is `v0.13.1`.

## What Failed

The failure was in the static release-candidate checker:

```bash
python scripts/check_release_candidate.py
```

After the documented local test workflow ran:

```bash
./scripts/run_tests.sh
```

the checker scanned the full local working directory and treated generated
runtime/test artifacts as package hygiene violations.

Observed failure classes:

```text
forbidden cache/runtime path present: .pytest_cache
forbidden local file present: .coverage
forbidden cache/runtime path present: .ruff_cache
forbidden cache/runtime path present: .venv
forbidden cache/runtime path present: reports/.mypy_cache
forbidden cache/runtime path present: src/cornerstone/__pycache__
forbidden compiled Python file present: *.pyc
```

This was a release tooling blocker. It was not a backend product behavior
failure.

## Root Cause

`scripts/check_release_candidate.py` mixed two different checks:

1. Release package hygiene: the shipped ZIP must not contain local caches,
   virtualenvs, `.env`, coverage files, or compiled Python files.
2. Local checkout hygiene after verification: the local checkout will naturally
   contain `.venv`, pytest cache, Ruff cache, `.coverage`, mypy cache, and
   `__pycache__` after running the documented verification workflow.

Because the checker used `ROOT.rglob("*")` against the local checkout, it failed
after normal verification created local runtime artifacts.

## What Changed in v0.13.1

### Version metadata

Updated package/runtime version from `0.13.0` to `0.13.1`:

```text
pyproject.toml
src/cornerstone/__init__.py
```

### Release checker

Updated:

```text
scripts/check_release_candidate.py
```

The checker now allows local-only generated artifacts when scanning the checkout:

```text
.venv
.pytest_cache
.ruff_cache
.mypy_cache
__pycache__
.coverage
```

It still rejects sensitive or packaged artifacts where relevant:

```text
.env
.pyc
.pyo
forbidden cache/runtime paths outside the local-runtime allowlist
```

### Regression test

Updated:

```text
tests/unit/test_release_candidate_docs.py
```

Added a regression test that imports `scripts/check_release_candidate.py`, runs
`check_package_hygiene(errors)`, and asserts that the current post-test local
checkout does not fail on generated runtime artifacts.

### Documentation

Updated:

```text
README.md
docs/release/backend-operator-runbook.md
docs/27-backend-release-candidate-v0.13.0.md
```

The operator runbook now expects `/healthz` version `0.13.1`.
