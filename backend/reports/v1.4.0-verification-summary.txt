## Environment limitations

The sandbox Python environment used for this packaging run does not have SQLAlchemy installed.
Persistent-store integration tests and Alembic offline SQL generation therefore could not be fully executed in this environment.

The attempted full ontology extraction integration test is recorded in:

```text
reports/pytest-ontology-extraction-v1.4.0.txt
```

It shows the in-memory tests passed before the persistent-store test failed at fixture setup due to:

```text
ModuleNotFoundError: No module named 'sqlalchemy'
```

The attempted ontology/manual integration test is recorded in:

```text
reports/pytest-ontology-manual-v1.4.0.txt
```

It similarly shows in-memory tests passed before a SQLAlchemy-backed persistence test failed at fixture setup.

The Python process also emitted a spreadsheet runtime warmup traceback to stderr during some normal startup paths. It did not change successful command exit codes. One direct pytest invocation for the release-doc unit test hung in this sandbox, so the same assertions were executed with a minimal python -S verifier and recorded in reports/pytest-release-docs-v1.4.0.txt.
