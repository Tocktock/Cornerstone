Cornerstone v1.5.0 Verification Summary
=======================================

Release theme
-------------
Ontology Candidate Review Workflow.

Goal
----
Implement the human review layer that lets authorized reviewers edit, approve, reject, or merge ConceptCandidates and RelationCandidates created by v1.4.0 extraction runs.

Trust boundary verified
-----------------------
- LLM/extractor output remains candidate-only.
- Only pending candidates can be reviewed.
- Reviewer authorization is enforced in production mode.
- Candidate approval promotes through the existing officialization gates.
- Candidate rejection creates no official Concept or ConceptRelation.
- Candidate merge updates an existing Concept or ConceptRelation and preserves evidence gates.
- Unsupported candidates cannot enter the official graph.

Manual proof behavior
---------------------
The v1.5.0 integration test exercises the settlement review loop:
manual uploaded settlement text -> EvidenceFragments -> reviewed evidence -> ontology extraction -> ConceptCandidates and RelationCandidates -> candidate approval/merge/reject -> official Concept/Relation creation through officialization gates -> depth-1 official graph includes approved RelationCandidate edge.
