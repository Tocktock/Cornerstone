## Slice Contract

Goal:

- Make `cornerstone scenario gate <report> --json` reject a VS3 local-dev assurance report when `scenario_run_id` or `trace_id` are consistently changed away from the current source-tree-bound identity.

In scope:

- Top-level `scenario_run_id`
- Top-level `trace_id`
- `traceability.scenario_run_id`
- `traceability.trace_id`
- Per-row `scenario_run_id`
- Per-row `trace_id`
- Native JSON gate output and focused regression coverage

Out of scope:

- RequestContext, RLS, OPA, egress, ConnectorHub live/provider flows, Tool SDK, signed registry, Agent Pack activation, and all VS3 human evidence gates.

Expected trace identity:

- `scenario_run_id=scenario-run:vs3-onprem-trusted-extension:<verified_source_worktree_hash first 16 chars>`
- `trace_id=trace:vs3-onprem-trusted-extension:<verified_source_worktree_hash first 16 chars>`

## Change Summary

Changed:

- `packages/cornerstone_cli/main.py`
  - Requires `scenario_run_id` and `trace_id` to be derived from `source_tree.verified_source_worktree_hash`.
  - Applies the check to top-level report metadata, `traceability`, and every scenario row.
  - Exposes `expected_trace_identity`, `actual_trace_identity`, and `actual_traceability_trace_identity` in `traceability_validation`.
  - Preserves existing `CS_VS3_TRACEABILITY_METADATA_MISSING` failure surface for traceability metadata issues.
- `tests/scenario/test_scaffold_cli.py`
  - Adds `test_vs3_scenario_gate_rejects_local_dev_claim_with_trace_identity_mismatch`.
  - Mutates `scenario_run_id` and `trace_id` consistently across top-level, traceability, and every row.
  - Asserts unrelated identity, reconciliation, coverage, claim-boundary, component-proof, ref, and transcript validators remain green.
