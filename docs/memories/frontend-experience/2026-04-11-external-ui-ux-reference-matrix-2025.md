## Reference flow sheets

### Dropbox Brand

Evidence:
- [desktop capture](../../../output/benchmarks/external-ui-ux-2025/references/dropbox-brand/desktop/landing.png)
- [mobile capture](../../../output/benchmarks/external-ui-ux-2025/references/dropbox-brand/mobile/landing.png)

CSS and visual grammar:
- huge typographic scale with very light framing
- blue-on-white palette uses brand color as structure, not decoration
- grid lines are visible enough to imply system logic without becoming chrome
- spacing is generous and uses whitespace as the primary hierarchy tool

UI structure:
- minimal top chrome
- one dominant message block
- system identity comes from grid, type, and crop rather than repeated cards

Actions and gestures:
- single obvious scroll cue
- almost no competing controls in first view
- interaction posture is calm rather than dense

Motion and transition behavior:
- public evidence suggests restrained reveal motion rather than constant animation
- route or section continuity appears to come from grid persistence and crop changes

Data representation:
- almost none in the first view
- the key lesson is how to let one artifact own the viewport before metadata appears

Operational fit:
- `adopt`: generous first-view hierarchy, structural grid hints, low-chrome framing
- `adapt`: strong brand-color usage must become muted editorial emphasis in Cornerstone
- `reject`: abstract brand-first messaging that delays access to the actual knowledge artifact
