# Cornerstone Backend v1.0.0 Release Notes

## Release Type

Backend MVP release.

## Release Basis

The `v1.0.0` tag should be created from the verified `v1.0.0-rc.1` release candidate only after:

- Clean checkout verification passed.
- Live PostgreSQL verification passed with zero skipped tests.
- Live Notion E2E verification passed.
- Product-loop proof passed.
- Safety negative checks passed.
- Secret scan passed.
- Human release sign-off is completed in `docs/release/v1.0.0-rc.1-human-signoff.md`.

## Proven Backend MVP Loop

```text
Live PostgreSQL
→ live Notion page
→ Artifact
→ EvidenceFragment
→ evidence review
→ official Concept
→ grounded context response
→ evaluation result
→ grounded_context_task_success_rate
```

## Known Limitations

- Notion page ingestion is included.
- Notion database/data_source ingestion is intentionally deferred.
- Slack, Google Docs, and GitHub connectors are post-1.0 work.
- Frontend/UI is not part of this backend release.
- Advanced vector retrieval/ranking is deferred.
- Enterprise SSO/RBAC is deferred.
- Production KMS/secret-manager integration remains a release-owner deployment decision.

## Final Release Gate

Before deploying, rerun:

```bash
./scripts/run_tests.sh
python scripts/check_release_candidate.py
python -m mypy src --show-error-codes --no-color-output --no-incremental
python scripts/run_live_postgres_tests.py --min-passed 5
PYTHONPATH=src python scripts/run_live_notion_e2e.py
```
